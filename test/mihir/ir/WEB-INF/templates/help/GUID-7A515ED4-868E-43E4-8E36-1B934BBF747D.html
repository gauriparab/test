Reanalyze a run

If a sequencing run fails to meet or exceed any QC metric defined in the assay, the QC Status in the Results Report screen will be flagged as failed (see View the Results Report). The run can be reanalyzed after relaxing the stringency of the failed metric.

To reanalyze the data from a run, a Manager or Administrator must first create a Reanalysis assay from the original assay, in which the QC metric that failed has been adjusted (see Create a Reanalysis Assay).

Note:

  • Managers and Administrators can reanalyze a sequencing run if the run completed without any critical alarms or errors. If the run aborted or produced major alarms or errors, the run cannot be reanalyzed.

  • QC metrics at the limits of stringency cannot be relaxed further. The sample library must be resequenced or a new library prepared.

To reanalyze a run:

  1. In the Data tab > Completed Runs & Results Screen , locate the sample, then click on the number in the Results column to the view the Results List screen.

    In the Results List screen, click View Result for the run that failed.

  2. Under the Actions column, the Reanalyze button will be active if a Reanalysis assay has been created. Click Reanalyze.
  3. In the Reanalyze dialog, select the Reanalysis assay from the dropdown list.
  4. Enter a name for the Results Report in the Report Name field, then click Reanalyze.

    The progress of the reanalysis can be followed under the Run Status column in the Results List screen. When reanalysis is done, the report name will appear in the Results column.
