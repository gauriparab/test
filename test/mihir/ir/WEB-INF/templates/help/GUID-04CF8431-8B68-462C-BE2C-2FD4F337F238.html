Data flow

During an Ion PGM™ Dx Sequencer run, sequence data files are transferred to the Ion PGM™ Torrent Server via a network cable in a process controlled by the ionCrawler service. After the data from the initial flows on the sequencer are available on the server, the Torrent Suite™ Assay Development Software begins processing the data, producing the <1>.wells file. Basecalling is performed on the <1>.wells file data, producing an unmapped BAM (uBAM) file. Subsequent analysis produces mapped reads (BAM) and variant calls (VCF) files.
