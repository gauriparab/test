Format the Analysis Status API

Note:

All request and response files will be in XML format only.

  1. To format the Analysis Status API, set the:

    • URL to: http://{reporterDx_ip}:{port}/ir/lims/get-plan-status?planName={plan_ name}

    • Header to: username={ user name}, password={user password}, Content-Type= application/xml

    • Query Parameter: LIMS should send a query parameter with name 'plan_name'. This should contain actual Planned Run name for which status will be returned.

    • Body: not required

    • Request Type = GET

  2. Complete the input fields:

    Field

    Description

    PlannedRunName

    This field will contain Planned Run name. It must be a single node.

    PlanShortCode

    This field will contain information about the generated run short code. It must be a single node.

    LibraryPrepResult

    This will hold status and output path information for each library prep. This may be multiple nodes.

    @status

    This will hold analysis status of library prep for a given Planned Run.

    @outputPath

    This will have the output path where analysis results will be stored. LIMS should have access to this path.

    @specimenID

    This will have the Sample ID for which library prep was created.

    LogMessages

    This will be unique node. It will contain Message nodes mentioned below.

    Message

    This will contain failure messages. There will be no message in case of success. This can be multiple nodes.
