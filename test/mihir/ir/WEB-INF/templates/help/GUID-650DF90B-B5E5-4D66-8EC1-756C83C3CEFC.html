Create a Reanalysis assay

If a sequencing run fails to meet or exceed any QC metric defined in the assay, the QC Status in the Results Report screen will be flagged as failed. You can find the metric that failed in QC Report tab in the Results Report screen (see View the Results Report).

The run can be reanalyzed after reducing the stringency of the failed metric. To do so, a Manager or Administrator must create a Reanalysis assay based on the original assay.

Note:

  • Managers and Administrators can reanalyze a sequencing run if the run completed without any critical alarms or errors. If the run aborted or produced major alarms or errors, the run cannot be reanalyzed.

  • QC metrics at the limits of stringency cannot be relaxed further. The sample library must be resequenced or a new library prepared.

Before creating the Reanalysis assay, identify the QC metric that failed in the QC Report tab in the Results Report screen.

  1. In the Assay tab > Manage Assays screen , locate the original assay used in the run. In the Actions column, click Reanalysis.

    The Create Assay screen screen will open to the QC page.

  2. Click on Library QC and Run QC to toggle between the parameters.
  3. Enter a new value for the QC metric that failed in the appropriate field. Click Reset All to reset all parameters to minimum requirements. Click Next.
  4. Adjust any parameters as desired, then click Next.
  5. Enter a name for the Reanalysis assay, and a description if desired.
  6. Click Save.

    The new assay will appear as a number under the Reanalysis Assay column in the Manage Assays screen. The number indicates how many Reanalysis assays have been created from the original assay.
