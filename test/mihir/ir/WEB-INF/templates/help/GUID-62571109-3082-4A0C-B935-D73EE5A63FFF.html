Variants tab

The Variants tab in the Results Report screen displays a table of variant results. The table has a different configuration and contains different columns depending on the assay type. In some configurations, the table is split into subtabs. Column descriptions are provided below.

The Summary subtab reports whether any Variants, Gene Fusions, or CNVs are detected. If present, the specific variants are listed.

Column

Description

Gene/Gene Symbol

The gene where the variant is located. The value is the information present in the last column of your target BED file. If the value is absent then the variant caller labels these positions as region#1…region#N .

HotSpot ID

The name of the hotspot as defined in the BED file (see Manage hotspot files).

Test Result

The determination of the assay for the given variant.

Locus

The allele position in the reference genome.

Type

The type of variation detected: SNV (single nucleotide variation)/MNV (multi-nucleotide variation), INS (insertion), or DEL (deletion).

Genotype

The genotype at the locus.

Ref

The reference base(s).

Var

The variant base(s).

Var Freq

The most likely frequency of the variant allele (see Note below).

Quality score

Quality score is the relative probability of either the "reference" hypothesis interval [0,cutoff), or the "variant" hypothesis interval [cutoff,1], Phred-scaled (‑10*log10). It is a measure of how strong the evidence for the variant call is. A higher score means more evidence for the call. The quality scores are capped at 100.

Cov

The number of reads covering the position after down-sampling.

Classification

The list of names from the associated classification sets.

Gene Isoform

The symbol of the gene isoform.

Driver Gene

The gene that regulates expression of the gene fusion isoform.

Type

The type of variation detected (e.g., Fusion).

Read Count

The number of valid reads aligned to specific isoform sequence.

Normalized Read Count

The ratio of read count and total fusion panel mapped reads.

Imbalance Score

The score calculated by subtracting the number of 5' reads from the number of 3' reads, and dividing the result by the sum of all expression control reads.

Copy Number

The copy number as determined by the software algorithm.

CNV Confidence

The confidence interval for the copy number estimate at 5% and 95% confidence.

# Amplicons

The number of amplicons associated with the CNV ID minus the amplicons tagged as being outliers.

Note:

The variant frequency reported in the run report is the most likely variant frequency in the reads after corrections are made for probable errors. The variant caller software uses this corrected frequency and the uncertainty in the observations to calculate the probability that the variant frequency in the sample falls within defined intervals that can be set by the user. The user sets a threshold frequency for deciding between genotypes: this defines intervals [0,c)[c,1-c)[1-c,1] corresponding to the three diploid genotypes (reference, heterozygous, homozygous) respectively. “c” here is the minimum variant frequency set during the creation of the assay. The software returns the genotype corresponding to the interval with the highest probability of containing the variant frequency in the sample. In most cases, the reads are highly accurate and do not have a high probability of error. In such cases, the observed frequency and the most likely frequency are similar, and are contained within a single interval leading to a high quality genotype. In some cases, if very few reads are observed, or the error rate is high, the observed counts and the most likely counts can be different, and the uncertainty in the real sample frequency can be high. When this happens, it is often impossible to exclude a heterozygous population in the sample, and the genotype is assigned to be heterozygous as a significant portion of the probability falls in that interval.
